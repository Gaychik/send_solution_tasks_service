Преподаватель загружает список студентов в формате "Имя-Фамилия" через веб-форму, также указывает название группы и свой Telegram chat_id.

Система генерирует уникальные логины и пароли для каждого студента.
Логины строятся на основе имени и фамилии (например, ivan.ivanov).
Все студенты привязываются к одной группе, а у группы сохраняется chat_id преподавателя.
Студенты входят в систему с помощью своего логина и пароля.

При отправке сообщения студентом бот на основе привязки группы к преподавателю отправляет сообщение в нужный Telegram-чат, прикрепляя к нему текст и файлы, если они были загружены.

Маршруты /teacher и /student:

Эти маршруты возвращают HTML-страницы для преподавателя и студента.
/teacher — страница для загрузки списка студентов и информации о группе.
/student — страница для входа студента и отправки сообщений.
Шаблоны Jinja2:

Используем шаблоны для рендеринга HTML.
Преподаватель загружает список студентов, а студенты могут отправлять сообщения после входа в систему.
После входа студента:

Система рендерит ту же страницу с полями для отправки сообщений, включая скрытые поля для имени и фамилии студента.
